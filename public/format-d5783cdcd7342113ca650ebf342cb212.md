# 文档格式

本文档基于 [Google 文档风格指南](https://google.github.io/styleguide/docguide/style.html)，适用于 Git 文档、飞书文档等所有 Markdown 写作场景。

可以直接将本文档作为提示词（system prompt）提供给 AI，要求其按照此规范输出文档。

## 基本假设

AI 写作时容易出现以下问题：

- 格式过度：滥用分隔线、表格、加粗装饰内容
- 结构过度：任何内容都用列表或表格组织
- 冗余：使用"非常"、"简单的"等修饰词
- 歧义：同一概念用不同名称

## 写作原则

- 删：删除不必要的格式元素，优先用段落和标题
- 简：能用列表就不表格，能用文字就不列表
- 少：全文格式元素（分隔线、表格、加粗）尽量少
- 一：同一概念全程使用相同名称

## 结构规范

### 标题层级

- 最多使用三级标题（`#` / `##` / `###`）
- 章节间用空行分隔
- 一级标题仅文档标题使用一次

## 标题规范

### 层级结构

- 最多使用三级标题（`#` / `##` / `###`）
- 避免在标题中使用标点符号
- 标题应简洁，明确概括内容

### 大小写规范

使用句子大小写（Sentence case），即标题首字母大写，其余小写（专有名词除外）。

## 分隔线规范

分隔线（`---`）用于划分文档主要部分。

### 使用原则

- 优先用空行+标题区分章节
- 分隔线仅用于重要划分点
- 全文最多使用 3 处

### 使用场景

可使用分隔线的场景：
- 文档顶部元信息与正文分隔
- 章节过多时划分大区块（2-3处）
- 按工作流程分区（如收集区、加工区、执行区）

应避免的场景：
- 每个章节间重复使用
- 作为装饰或视觉填充

## 列表规范

### 使用原则

无序列表（`-`）适用于：
- 并列的多个要点
- 不分先后顺序的内容
- 短小的条目

有序列表（`1.`）适用于：
- 有明确顺序的步骤
- 需要编号的操作流程
- 排名或优先级

应避免的场景：
- 列表项过长（超过两行）
- 嵌套超过 2 级
- 滥用列表代替段落

### 无序列表

- 使用 `-` 作为列表标记
- 列表项以分号结尾或使用完整句
- 保持列表项风格一致

### 有序列表

1. 步骤类内容使用有序列表
2. 确保编号连续
3. 每个步骤完整表达

### 嵌套列表

- 缩进使用 2 个空格
- 嵌套层级不超过 2 级

## 代码块

### 语言标注

必须标注代码语言类型：

\`\`\`bash
git status
\`\`\`

\`\`\`python
def hello():
    print("Hello")
\`\`\`

### 行内代码

使用反引号包裹：`variable`、`function()`、`file.md`

## 表格规范

### 使用原则

表格用于呈现多维度、需对比的数据。

应使用表格的场景：
- 需要横向对比多个项目
- 数据具有明确的列属性
- 信息结构化为行记录

应避免的场景：
- 仅是简单的名词-定义对应（用列表代替）
- 两列且无对比需求
- 单元格内容过长

### 基本格式

| 列1 | 列2 | 列3 |
|------|------|------|
| 内容 | 内容 | 内容 |

### 规范

- 表头加粗
- 列对齐使用冒号（`:--`、`:--:`、`--:`）
- 内容简洁，避免单元格过长

## 链接规范

### 外部链接

[链接文本](https://example.com)

### 内部链接

使用相对路径：`[文档](./docs/guide.md)`

## 加粗规范

### 使用原则

加粗用于强调关键词，避免过度使用。

应使用加粗的场景：
- 首次定义关键术语
- 强调重要的操作或警告
- 引导注意力到关键信息

应避免的场景：
- 标记所有术语名词
- 连续多个加粗
- 在列表项内部使用

### 块引用

> 使用块引用标注重要提示、警告或引用内容

## 引号规范

### 使用原则

引号用于引用内容、特定术语或强调，避免过度使用。

应使用引号的场景：
- 直接引用他人的话语或文本
- 引用特定术语或概念名称（首次定义时）
- 强调某个词汇的特殊含义或用法
- 标注按钮、菜单项等界面元素名称

应避免的场景：
- 包裹所有术语名词
- 用于普通词汇的强调（用加粗代替）
- 在列表项或标题中频繁使用
- 包裹整个句子或段落作为"引用"

### 引号类型

英文文档使用双引号：
- 直接引用："This is a quote"
- 术语引用："variable name"

中文文档使用中文引号：
- 直接引用：「这是引用内容」或「这是引用内容」
- 术语引用：「变量名」

### 示例

正确使用：
- 文档中首次出现术语「微服务」时，用引号标注
- 引用用户反馈：「这个功能很好用」

错误使用：
- 把所有术语都加上引号（如「系统」、「模块」、「接口」）
- 用引号强调整段文字：「这段话很重要」

### 替代方案

- 普通强调使用加粗：**关键信息**
- 术语标记使用行内代码：`variable`
- 大段引用使用块引用：> 引用内容
