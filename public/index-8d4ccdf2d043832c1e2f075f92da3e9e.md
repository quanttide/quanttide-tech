# 数据工程蓝图

本文档定义数据清洗项目的数据模型、处理流程、质量规范。

详见 [docs/index.md](../index.md) 了解业务背景和业务知识。

## 数据目录

### 原始数据模型

#### 产量数据 (`raw/半年产量数据/`)

核心生产记录，每行代表一个员工某天完成某工序的数量。

| 字段 | 类型 | 业务含义 |
|------|------|----------|
| 款号 | string | 款式编号（需清洗） |
| 工序序号 | int | 工序编号 |
| 工序 | string | 工序名称（员工填报，可能与标准名不同） |
| 员工 | string | 员工姓名 |
| 日期 | date | 生产日期 |
| 产量 | int | 当日完成数量 |

#### 工序表 (`raw/工序表/`)

标准工序定义，每款号一张表，定义该款的所有工序。

| 字段 | 类型 | 业务含义 |
|------|------|----------|
| 款号 | string | 款式编号（从表头提取） |
| 工序号 | int | 工序编号 |
| 工序名称 | string | 标准工序名称（两个版本：表1/表2） |
| 机器类型 | string | 所需设备 |
| 基础时间 | float | 标准工时（秒） |
| GST时间 | float | GST标准时间 |
| 工序等级 | string | 难度等级 |
| 小时目标 | int | 每小时目标产量 |
| 部位 | string | 所属部位 |

#### 返工数据 (`raw/返工数量/`)

返工记录，每行代表一个员工某工序的返工数量。

| 字段 | 类型 | 业务含义 |
|------|------|----------|
| 员工 | string | 员工姓名 |
| 工序名称 | string | 工序名称 |
| 工序序号 | int | 工序编号 |
| 日期 | date | 返工日期 |
| 返工数量 | int | 返工件数 |

#### 考勤数据 (`raw/考勤/`)

员工出勤记录，宽表结构（日期为列）。

| 字段 | 类型 | 业务含义 |
|------|------|----------|
| 姓名 | string | 员工姓名 |
| 11日、12日... | string | 每日出勤时间（如 "10h"） |

### 最终数据模型

#### 合并宽表 (`final/产量数据_工序_返工_考勤_合并.xlsx`)

| 字段 | 类型 | 业务含义 | 来源 |
|------|------|----------|------|
| 款号 | string | 原始款号 | 产量数据 |
| 清洗后款号 | string | 标准化款号 | 款号清洗 |
| 工序序号 | int | 工序编号 | 产量数据 |
| 工序 | string | 原始工序名 | 产量数据 |
| 工序名称_表1 | string | 标准工序名（版本1） | 工序表 |
| 工序名称_表2 | string | 标准工序名（版本2） | 工序表 |
| 机器类型 | string | 所需设备 | 工序表 |
| 基础时间 | float | 标准工时（秒） | 工序表 |
| GST时间 | float | GST标准时间 | 工序表 |
| 工序等级 | string | 难度等级 | 工序表 |
| 小时目标 | int | 目标产量 | 工序表 |
| 部位 | string | 所属部位 | 工序表 |
| 员工 | string | 员工姓名 | 产量数据 |
| 日期 | date | 生产日期 | 产量数据 |
| 产量 | int | 当日产量 | 产量数据 |
| 返工数量 | int | 当日返工件数 | 返工表 |
| 出勤时间 | string | 当日出勤时长 | 考勤表 |

## 数据处理流程

### 流程图

```
Step 1: 产量数据 + 工序表 → 款号清洗 + 工序匹配
    ↓
Step 2: + 返工数据 → 返工关联
    ↓
Step 3: + 考勤数据 → 考勤关联
    ↓
最终宽表
```

### Step 1: 款号清洗与工序匹配

**目标**：为每条产量记录匹配标准工序属性。

**关联键**：`[清洗后款号, 工序序号]`

**处理逻辑**：
1. 款号清洗：应用规则库（去后缀、修正拼写、同款映射）
2. 工序匹配：以 `[清洗后款号, 工序序号]` 左连接工序表
3. 未匹配记录输出到 `未匹配款式工序列表.xlsx`

### Step 2: 返工数据关联

**目标**：为每条产量记录补充返工数量。

**关联键**：`[员工, 工序名称, 日期]`

### Step 3: 考勤数据关联

**目标**：为每条产量记录补充出勤时间。

**关联键**：`[员工, 日期]`

**处理逻辑**：
1. 宽表转长表：考勤数据从日期列转为记录行
2. 日期解析：从 Sheet 名称提取月份（如"3月"），假设年份为 2025

## 数据质量规范

### 款号清洗规则

| 规则类型 | 示例 |
|----------|------|
| 去后缀词 | `16A0509-12-28含尾部` → `16A0509` |
| 去描述词 | `12C0933加单` → `12C0933` |
| 修正拼写 | `15CO156` → `15C0156` |
| 格式修正 | `31110011` → `3111-0011` |
| 同款映射 | `67C0015` → `12C1470` |

### 工序匹配原则

- 使用模糊匹配，容忍工序名称差异
- 排除非生产工序：裁床、尾部、大烫、包装、检验

### 质量指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 款号匹配率 | ≥ 90% | 成功匹配工序表的记录占比 |
| 工序匹配率 | ≥ 80% | 成功匹配工序属性的记录占比 |
| 考勤匹配率 | ≥ 85% | 成功关联考勤的记录占比 |

### 异常处理

- 未匹配款号：输出到单独文件，供人工检查
- 未匹配工序：工序表字段为空，但保留原始工序名
- 未匹配考勤：出勤时间字段为空
