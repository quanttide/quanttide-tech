# 量潮数字资产云产品愿景

## 展示数字资产全景图

从微观上来看，我们使用大量自定义的数字资产承载了知识与流程。以量潮数字资产云（quanttide/qtcloud-asset）为例，产品研发资产包括文档（docs）、示例（examples）、代码（src）等，文档资产又包括商业需求文档（brd）、产品需求文档（prd）、交互设计文档（ixd）、架构设计文档（add）、质量保证文档（qa）、用户文档（user）等。


更复杂的是，这些资产之间有着复杂的信息流转关系。虽然大体上我们可以按照一个线性关系排序，但是也不排除经常会有非常规流程出现。可以说有多少资产就有多少对应关系。


请读者动用自己的空间想象力，想象有一个软件展示了所有数字资产和他们之间的关联关系。展示这样一个知识图谱，需要对应的知识建模、知识抽取和可视化技术。


如果把数字资产看成数据，那么这本质上是一个 Data Fabric 架构。只不过这里我们展示的是企业有业务意义的数字资产关系。


## 为 Agent Harness 提供交付约束


另外一个更根本的目标是约束 AI。


人是可以被隐形约束的，只是相比之下隐形约束可靠性和可传递性弱一些。但是 AI 必须要显性的可以被规则引擎验证的约束。


Agent Harness 时代下的人机分工将会发生根本的变化。人不可能也不需要控制 AI 的行为，但是人一定需要能够验收人类的结果。那么约定人机之间的交付物就是不可避免的。


我们如何约束纷繁复杂的工作流产生的纷繁复杂的数字交付物？我们必然需要一个专门约束 Harness 的 Harness。在量潮云的领域划分体系下，承载这个任务的主要是数字资产云。


## 助力数字资产定价


我们这种软件科技公司的资产构成有很大一部分是无法被定价的数字资产、品牌资产、知识产权等等。在量潮云的规划中，数字资产云、品牌云、知识产权云分别将会承担这些领域的工作。


缺乏定价依据的根本原因是缺乏市场和价格。所以我们首先要想办法让市场和价格产生。那么剩下的问题就是一个叫做“资产定价”的金融学分支的工作了。


价格从哪里来？各种各样的交易过程。我们和客户的交易，这在商务领域的报价子领域。我们和供应商的交易，这在供应链领域的采购子领域。我们和员工的交易，这在人力资源领域的薪酬和绩效子领域。


理论上我们可以从这些价格中，通过某些科学合理的建模方式，得到一个相对可靠的定价，从而为企业未来的各类定价决策提供有效的依据。
