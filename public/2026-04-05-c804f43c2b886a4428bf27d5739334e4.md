### 正式描述

> 根据原始描述提炼基础设施编排需求

1. 使用环境：仅使用手机和网吧电脑进行编程。
2. 云服务商：阿里云，使用企业账号，采用按量付费模式。
3. 核心部署目标：
   · 部署一个 Open Code Server（如 code-server）服务。
   · 部署一个 New API 服务，用于统一管理各种 AI 的调用（例如聚合 OpenAI、Claude、国产模型等 API）。
4. 存储要求：后端连接云存储，优先使用对象存储（OSS），亦可支持块存储或文件存储。
5. 网络与代码托管：能连接国内代码托管平台 GitLink，并支持 Git 操作。
6. 管理方式：
   · 所有配置容器化，通过启动容器即可运行环境。
   · 使用 Terraform + Dockerfile + README 作为最小基础设施代码（IaC）组合。
   · 单独设立一个资源组进行独立管理，方便实验。
7. AI 维护期望：基础设施代码应相对静态、稳定，以便 AI 能够帮助维护和编辑，避免过于动态的配置导致不稳定。


## OpenCode-Cloud

### 部署方法

单容器包含（Docker in Docker），只需要Docker即可部署。唯一的依赖是大模型 API。

持久化存储只能挂载 NAS 文件系统，不支持 S3 协议。

Git 已经内置，不需要额外配置。


## 实验结果

阿里云控制台基本不可用，在云端无法从零初始化开发环境。

网吧电脑无法下载和使用Trae和OpenCode等，也无法通过本地安装实现。

Linux 和 macOS 都暂不可用，试试 Windows 备用机安装 OpenCode 是否可用。

也不是一无所获。这意味着在国内环境下搭建可用开发基础设施远比我们想象的困难，市场上甚至很难找到一个即开即用的工具。
