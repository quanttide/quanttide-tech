
# Qwen（性格：勤奋但有点神经质的理工科直男）

优：
- 代码编写；
- 数学解题
- 需要高频调用API的成本敏感型项目、中文场景的各类任务。

劣：
-网页端经常忽略上下文
-老是弹出很复杂的东西，明明我的需求很简单

# Gemini（性格：中等偏上的全科班长，中规中矩）

优：
- 啥都能干
劣：
- 但是都不精通，中规中矩

# Deepseek（性格：文科生 / 接地气的文案+日常助手）

优：
- 日常事务的处理，比如写邮件文案/总结；
- 角色扮演；
- 娱乐聊天
- 高性价比长文本

劣：
- 深度推理/硬核代码能力不足

# Claude（性格：极其认真负责的学霸学长，代码审查官）

优：
- 代码审查与调试；
- 日常办公场景，包括财务分析、课题研究，以及文档、电子表格和演示文稿的创建与处理；
- 根据上下文线索自主判断扩展思维的使用强度；
- 代码能力强、多文件重构、长上下文(1M)、深度推理

劣：
- 贵贵贵

# Grok（性格：嘴毒、爱搜、消息灵通的段子手理工男）

优：
- 搜索能力较强；
- 社交媒体洞察；
- 超长上下文：2000k tokens

劣：
- 深度思考能力一般

# MiMo（性格：铁直男理科生，细节控+代码设计狂魔）

优：
- 评审能力较好；
- 代码设计能力好;
- 理科男，一到代码等专业领域就会返回很详细，考虑细致;
-如果想自己搞一个龙虾，可以用Mimo，意图识别的蛮准确

劣：
- 没有“人文关怀”，回复的很直男，铁顺直;
- token价格较高、但在可承受范围内。

# 总结区-快速任务选型
（以下结论根据技术委员会成员个人体验得出，若有偏差概不负责【狗头保命】）
- 写代码 / 重构 / debug -> 首选 Claude > MiMo > Qwen
- 数学 / 硬核推理 -> Claude = MiMo > Qwen > DeepSeek
- 写中文材料 / 邮件 / 文案 / 总结 -> Deepseek > Qwen > Claude
- 需要最新实时信息 / 热点 / 搜东西 -> Grok > Gemini
- 多模态 / 图表 / 视频 / 长报告处理 -> Gemini > Claude
- 高频调用 / 成本敏感 / 大量中文 -> Qwen 或 Deepseek
- 要做很复杂的系统 / Agent / 意图识别要准 -> MiMo > Claude
- 啥都不知道先用哪个 -> Gemini（最不容易翻车）
