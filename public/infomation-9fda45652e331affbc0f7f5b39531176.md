# 组织信息透明化改革蓝图

## 背景

新型组织契约把个体责任从"服从指令即可免责"变成了"理解系统才能负责"，但公司只给了新契约文本，没有给理解系统的认知工具。团队新人们不理解自己的角色履职异常对公司发不出工资有什么影响，以及他们应该承担什么责任。

这是公司代表大会建设的重点。

---

## 任务清单

### 1. 绘制角色影响地图

明确每个人的每个角色对于结果的影响。

**目标**：让每位成员清楚自己的行为如何影响公司整体运营。

### 2. 定义责任与职权类别

明确责任类别：

| 类别 | 说明 |
|------|------|
| 执行责任 | 按时完成本职工作 |
| 预警责任 | 发现异常及时上报 |
| 修复责任 | 主动解决问题 |
| 反思责任 | 复盘改进流程 |

同时定义对应的职权类别。

### 3. 设计模拟经营工坊

以演习的形式展示某个岗位失职对公司有什么影响。

**形式**：模拟经营系统，让成员在安全环境中体验决策后果。

### 4. 章程手册化

把章程翻译成手册的日常决策红线，变成可以自检的问题。

**输出**：可操作的自检清单，而非抽象的章程条款。

---

## 案例：商务经理

**现状**：商务拓展职能基本上还是靠创始人给线索，或者吃现有人脉，商务部门没有主动找线索。对现有线索的处理质量也很差，实际上发挥的作用只是跑流程。

**改进方向**：
- 建立主动获客机制
- 提升线索处理质量
- 从流程执行者转变为价值创造者
