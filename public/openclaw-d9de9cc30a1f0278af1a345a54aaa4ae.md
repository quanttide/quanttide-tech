# OpenClaw

## 概述

OpenClaw 是个人 AI 助手，运行在本地设备上。

- **类型**：个人 AI 助手，运行在本地设备上
- **模式**：响应式、对话式、持续在线

## 常用命令

启动：openclaw gateway start

## 最佳实践

目前可以作为一个"值班秘书"在本地电脑开机运行时帮忙监管，方便在手机上、在室外监督，减少坐办公室的时间。最主要的作用是在手机上想工作可以去电脑直接验证，减少灵感验证的周期。

本地连接十分不稳定，把服务部署在云端，把主要依赖都移到云端，才可能可以稳定地上生产。后台要稳定常驻，依赖的组件要安全且开放。

OpenClaw 不太具备真正的长时工作能力，执行力相比于 Code cli 差得多，且更加不透明难以监管。如果想要用它，就需要设法合理监督。要多问它一些你知道的信息，看它掌握的和你掌握的是否一致。

由于OpenClaw无法有效使用opencode，所以其主要职能就只是记录到workspace里，方便在电脑上用opencode读取记录。
