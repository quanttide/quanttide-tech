# 执行日志格式说明

## 文件命名
- 每日执行日志文件命名格式为：`YYYY-MM-DD.md`
- 例如：`2026-04-08.md`

## 文件结构
每份执行日志应包含以下标题层级：

### 一级标题
```
# 日期 执行日志
```
- 日期格式：`YYYY-MM-DD`
- 例如：`# 2026-04-08 执行日志`

### 二级标题（固定栏目）
日志正文应包含以下五个二级标题，按顺序排列：
- `## 收集`
- `## 加工`
- `## 整理`
- `## 执行`
- `## 反思`

## 内容说明
- **收集**：记录当天收集到的信息、数据、资料等
- **加工**：对收集内容进行的初步处理、分析、转换等操作
- **整理**：对加工后的内容进行归纳、分类、结构化等工作
- **执行**：实际执行的任务、行动、实施步骤等
- **反思**：对当天工作的总结、经验教训、改进建议等

## 示例
```markdown
# 2026-04-08 执行日志

## 收集
- 收集了项目需求文档
- 下载了最新的技术资料

## 加工
- 分析了需求文档中的关键功能点
- 整理了技术资料的主要内容

## 整理
- 创建了需求功能清单
- 建立了技术知识库框架

## 执行
- 完成了模块A的初步设计
- 开发了接口B的基本功能

## 反思
- 时间管理需要改进，某些任务超时
- 下天计划更细致地分配时间块
```
