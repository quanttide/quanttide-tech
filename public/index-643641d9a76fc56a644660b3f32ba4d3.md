# 实训课

一对一定制实训教学服务。

## 教学计划

一、目标

做什么：

1. 学习使用Vibe Coding做生产级平台的方法，包括探索、重构、整合等。

2. 学习人机协同提高元认知能力的方法，包括观察、反思、记录等。

不做什么：

1. 成品不用符合设计预期。探索式编程就是在验证想法是不是具备可行性，使得后续研发少走弯路。

2. 老师也不预知答案，而是只大概知道方向。老师和学生一起以同事的身份探究。

二、洞察

Vibe Coding时代的程序员最不可替代的能力是元认知能力和维护代码不跑偏的能力。

三、愿景

学生可以掌握Vibe Coding写生产级应用的方法，并且能够独立总结属于自己的方法论。

四、项目

基础项目：qtadmin。
激励项目：各领域工具库和工程标准等。有额外激励，适合写简历里"参与量潮标准化体系建设"。

五、流程

洞察：反复重写的过程中不断精炼对业务需求的理解，加深理解AI怎么理解问题以掌握AI。

流程：

1. examples

2. tests

3. src

4. packages（可选）

六、任务

1. 议事型课堂讨论方案、制定计划。

2. 实现代码和 PR，被 review。

3. 工作档案记录过程 qtclass-profile-of-xxx。

七、进化

此方案可以标准化，既可以作为云业务的孵化机制，也可以作为课堂的实训机制，还可以作为数据业务的升级机制，实现产教融合的效果。
