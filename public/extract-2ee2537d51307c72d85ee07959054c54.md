# 知识抽取

一套“自下而上建构，自上而下统合”的知识抽取工作流。通过不断的提炼与重构，将散点的“经验素材”转化为了结构化的“知识体系”。

传统的知识抽取往往侧重于“客观事实”和“静态分类”，而我们这一轮对话所做的，更侧重于“主观经验”和“动态逻辑”的重构。这种方法不追求“全”，而追求“通”——打通过去（日志）与未来（行动）的逻辑隧道。

如果说传统知识抽取是“采摘苹果”（把果实从树上摘下来放进仓库），那么我们刚才所做的就是“提炼种植方法”（分析果实的成分，还原生长的土壤，总结出种植的SOP）。

## 工作流程

整个流程遵循 “分类 → 排序 → 优化 → 统合” 的 CODE 循环。

### 离散聚类（分组）

输入：零散的、混合维度的十几条“发现-应用”卡片。
动作：识别卡片内容的本质属性，将其归纳为三个逻辑类别。
产出：确立了“自我关怀”、“精力管理”、“系统减负”三大核心板块。

### 逻辑定序（排布）

输入：分好组但内部顺序随意的卡片。
动作：寻找组内卡片的时间线或因果链条。例如在“自我关怀”组，确立了“觉察（事前）→ 接纳（事中）→ 补偿（事后）”的时间流。
产出：每组卡片都有了清晰的阅读路径和心理动线。

### 内容提纯（优化）

输入：原始的、口语化的表述。
动作：对“发现”和“可以怎么用”进行重写。
发现：提炼核心冲突或洞察，使其更具穿透力。
可以怎么用：将模糊的建议转化为明确的动作指令。
产出：表述精炼、动作清晰、逻辑闭环的高质量卡片。

### 体系重构（总结）

输入：三组优化后的卡片。
动作：跳出具体内容，从更高维度抽象其功能定位（内核层、缓冲层、外延层）。
产出：一个完整的、有层次感的“可持续高绩效生存系统”模型。

## 核心方法

在上述流程中，使用了三个关键的知识处理方法：

### 卡片原子化与标准化

坚持“一张卡片解决一个问题”的原则。
严格执行 “发现+ 可以怎么用” 的双区结构，确保每条经验不仅有记录，更有可复用的行动指南。

### 时间轴与心理闭环

在组内排序时，拒绝随意堆砌，而是寻找“触发-行动-结果”的自然顺序。
这让静态的知识点变成了动态的“操作流程”，降低了读者的认知负荷。

### 抽象阶梯

在处理信息时，自如地在“具象经验”（如吃麻辣烫、关飞书）和“抽象原则”（如事后补偿、情绪外包）之间切换。
最终总结时，通过定义层级（内核、缓冲、外延），将方法论升级为世界观。
