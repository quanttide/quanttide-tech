# AGENTS.md - Agent 工作指南

## 文件命名规范

### 日志文件
- **位置**: `meta/journal/learn/`
- **格式**: `YYYY-MM-DD.md` (例如: `2026-03-18.md`)
- **说明**: 学习日志按日期命名，方便追踪和查找

### 通用规则
- 使用小写字母
- 单词之间用下划线 `_` 分隔 (如果需要)
- 日期格式: `YYYY-MM-DD`

## 学习日志规范

### 定义
学习日志是 **陈述型记忆** 在 **现在** 时间轴、**事件类** 分类下的原始记录。
它是主体的学习记录。meta 模块的主体是第二大脑自己。

### 结构
- **标题**: `YYYY-MM-DD 学习 [主题]`
- **学习目标**: 明确本次学习的目的
- **学习内容**: 详细的观察、记录、框架理解
- **学习收获**: 总结关键洞察
- **下一步行动**: 基于学习的行动计划

### 示例
参考 `meta/journal/learn/2026-03-18.md`。

## 工作原则

1. **最小干预**: 仅在用户明确请求时操作
2. **原子提交**: 每次提交独立完整
3. **验证优先**: 修改后检查文件命名和内容
4. **明确命名**: 严格按照规范命名文件
